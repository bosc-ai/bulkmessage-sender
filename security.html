Trust & Safety

Security at WaChat

How we protect your data and your customers' conversations.

🔐

Encryption in transit

All data between your browser, our servers, and Meta's API is encrypted with TLS 1.3. HTTP connections are automatically redirected to HTTPS.

🗄️

Encryption at rest

All stored data is encrypted with AES-256. Database backups are encrypted and stored in isolated storage with strict access controls.

🏗️

Infrastructure

WaChat runs on AWS Mumbai (ap-south-1) with failover to Singapore (ap-southeast-1). Infrastructure is isolated per tenant with separate database namespaces.

🔑

Access control

Role-based access controls (RBAC) limit what each team member can see and do. Principle of least privilege enforced throughout. All admin actions are logged.

📋

Audit logs

All platform events — logins, template changes, campaign sends, API calls — are logged with timestamps and user attribution. Logs are immutable and retained for 12 months.

🔒

Secret management

API keys, OAuth tokens, and webhook secrets are stored in a dedicated secrets manager, never in source code or logs. Keys are rotated on schedule.

Vulnerability disclosure

We welcome responsible disclosure of security vulnerabilities. If you discover a potential security issue in WaChat, email us at security@serves.in with:

  • A description of the vulnerability
  • Steps to reproduce
  • Potential impact assessment
  • Your contact information (optional)

We will acknowledge your report within 2 business days and aim to resolve critical issues within 7 days. We ask that you give us reasonable time to investigate before any public disclosure.

Application security

  • OWASP Top 10 mitigations applied throughout the application
  • SQL injection and XSS protection via parameterised queries and output encoding
  • CSRF tokens on all state-changing requests
  • Rate limiting on authentication endpoints
  • Multi-factor authentication (MFA) available for all accounts
  • Regular dependency security scanning

Data handling

  • Data is logically isolated per organisation — no cross-account access possible
  • WhatsApp message content stored only as long as needed for inbox operation
  • Employee access to production data requires justification and is logged
  • Subprocessors are contractually bound to security standards

Incident response

In the event of a security incident affecting your data, we will notify you within 72 hours of becoming aware, as required by applicable law. Notifications go to the primary email address on your WaChat account.

Compliance

  • India DPDP Act 2023 — data processing principles applied throughout
  • WhatsApp Business Platform Terms — all data use within permitted scope
  • GDPR-aligned practices for EU users

Security contact

Email: security@serves.in

Response time: 2 business days · Critical issues: same-day acknowledgment
